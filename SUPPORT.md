# Support

## Getting help

Search the [documentation](docs), [troubleshooting guide](docs/troubleshooting.md), and existing [GitHub issues](https://github.com/Lebyy/tracewhy/issues) first. If the problem remains, open a bug report with the issue template.

Include the TraceWhy version, operating system and architecture, whether the input is a recording directory or bundle, the smallest safe reproduction, and sanitized output. Do not upload a recording, JSON export, HTML report, raw trace, private path, command argument, or environment value until you have reviewed it.

Community support is best-effort. There is no guaranteed response time or private consulting channel.

## Security issues

Do not file a public issue for a vulnerability or redaction bypass. Follow the [security policy](SECURITY.md) and use GitHub private vulnerability reporting.

## Platform requests

Read the [platform support contract](docs/platform-support.md) before requesting a capture backend. A new backend must preserve evidence quality, containment, redaction, and reproducibility; merely launching the command on another operating system is not sufficient.
